# RUN_TEMPLATE

## 使用说明

* RUNMD 为日志模式（log），记录系统运行时/运维阶段的报错、警告与处置闭环。
* 本模板只用于运行阶段事件；编译、代码、依赖、测试错误应写入 ERRORMD。
* 每次新增 RUN 记录后，必须同步更新 `RUN_INDEX.md`。

## Metadata

* **修改者：** [执行器 Agent 名称，例如 Codex]
* **事件时间（UTC）：** [YYYY-MM-DD HH:MM]
* **事件名（Key）：** [稳定事件名，例如 API_HEALTHCHECK_TIMEOUT]
* **分类：** [DEPLOY / HEALTHCHECK / RESTART / ROLLBACK / INCIDENT / OBSERVABILITY / OTHER]
* **级别：** [FATAL / ERROR / WARNING / NOTICE / INFO / DEBUG]
* **影响范围：** [服务、节点、环境或任务范围]
* **状态：** [OPEN / RESOLVED / MITIGATED]
