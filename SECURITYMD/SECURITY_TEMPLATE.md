# SECURITY_TEMPLATE

## 使用说明

* SECURITYMD 为日志模式（log），仅记录“已发生攻击或已确认恶意行为”的安全事件。
* 本模板用于漏洞响应与安全处置闭环；日常安全建议不写入 SECURITYMD。
* 每次新增 SECURITY 记录后，必须同步更新 `SECURITY_INDEX.md`。
* 每次新增 SECURITY 记录时，必须通过 `Linkage` 明确联动 `ERRORMD` 或 `RUNMD`（至少一个）。

## Metadata

* **修改者：** [执行器 Agent 名称，例如 Codex]
* **事件名（Key）：** [稳定事件名，例如 LOGIN_BRUTEFORCE]
* **分类：** [ATTACK / INTRUSION / BRUTEFORCE / ABUSE / INCIDENT / OTHER]
* **级别：** [FATAL / ERROR / WARNING / NOTICE / INFO / DEBUG]
* **影响范围：** [服务、接口、节点、租户或数据范围]
* **状态：** [OPEN / RESOLVED / MITIGATED]
