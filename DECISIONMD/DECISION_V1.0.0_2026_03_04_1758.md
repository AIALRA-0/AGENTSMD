# DECISION_V1.0.0_2026_03_04_1758

## Metadata

* **修改者：** Codex
* **版本号：** V1.0.0

## Summary

* 确立 AGENTSMD 分部门治理与索引优先访问作为长期架构基线。

## Thought

* 当前文档体系若缺少稳定治理骨架，后续扩展会持续出现规则漂移与职责重叠。
* 重大决策、日常变更、事件日志必须分层管理，才能保障检索效率与审计可追溯性。
* 采用分部门治理可以明确每类信息归属，降低跨目录混写带来的维护成本。
* 未采纳“单文档承载全部约束”方案，因为该方案难以支持长期演进与索引级访问。
* 本次决策目标是让后续规则迭代都能在统一框架内增量完成。

## Action

* 规定所有后续约束变更必须先在对应部门落地，再通过索引暴露最新入口。
* 规定所有架构级调整必须先写 DECISION，再在 CHANGEMD 记录执行落地行为。
* 规定 AGENT 执行前必须按目录索引读取，不允许跳过索引直接写条目。
* 规定模板和索引字段以规则源统一治理，避免部门间字段失配。
* 本决策的执行落地需在 CHANGEMD 至少新增一条对应 CHANGE 记录。

## Observation

* 决策发布后，AGENTSMD 已形成“规则源-模板-索引-样本”四层可维护结构。
* 决策目标与当前目录状态一致，后续新增部门可按同一方法平滑接入。
* 主要风险在于执行不一致，需依赖脚本校验和索引同步持续约束。
* 后续应在每次规则演进后复盘 DECISION 与 CHANGE 的联动完整性。
* 当前决策可作为后续架构升级和规范收敛的基准起点。
