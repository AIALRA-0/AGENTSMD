# SECURITY_ERROR_LOGIN_BRUTEFORCE_2026_03_05_0055

## Metadata

* **修改者：** Codex
* **事件名（Key）：** LOGIN_BRUTEFORCE
* **分类：** BRUTEFORCE
* **级别：** ERROR
* **影响范围：** service-agentmd-api 登录接口与认证网关
* **状态：** MITIGATED

## Summary

* 登录接口遭受暴力破解尝试（峰值 420 req/s），处置后异常流量下降 81%，服务恢复正常

## Thought

* 暴力尝试已触发告警阈值（失败率 >30%），需立即止损避免认证服务资源耗尽
* 风险边界：若持续放大，可能导致正常用户登录失败或 DoS 级联
* 处置目标：3 分钟内完成封禁与限流，10 分钟内完成记录与索引更新

## Action

* 定位 /login 端点失败率异常飙升至 42.7%，来源 IP 集中于 3 个 /24 网段
* 执行源 IP 聚合分析，Top3 攻击源请求频率达 420 req/s
* 下发 WAF 规则即时封禁 Top10 IP 及高频 /24 网段
* 启用登录接口单 IP 限流（60 req/min）并强制升级二次验证码
* 回放认证日志，确认正常用户未被误伤

## Observation

* 处置前登录失败率峰值 42.7%，处置后 5 分钟内降至 2.1%
* 异常流量下降 81%，正常登录成功率恢复至 99.8% 基线
* 当前无持续 5xx 错误，但需观察每日 02:00~04:00 峰值是否出现源 IP 轮换

## Root Cause

* 登录接口缺少分层风控（IP 信誉 + 行为评分），导致暴力尝试短时穿透首层防护

## Fix

* 即时封禁高风险 IP + 启用单 IP 限流 + 强制二次验证码挑战

## Prevention

* 将登录失败聚合告警阈值前移（连续 20 次失败触发自动封禁建议）
* 建立登录安全事件处置手册（证据抽取 → 封禁 → 限流 → 误伤回查），纳入 RUNBOOKSMD
* 引入设备指纹 + 行为评分分层风控，目标将暴力成功率降至 <5%
