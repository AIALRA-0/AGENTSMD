# Security Policy

## Supported Versions

The `main` branch is the supported baseline.

## Reporting a Vulnerability

Please do not open public issues for undisclosed vulnerabilities.

Report details privately to maintainers with:

- Affected paths/modules
- Reproduction steps
- Impact assessment
- Suggested mitigation (optional)

We will acknowledge the report within 72 hours and coordinate remediation and disclosure timeline.

## Handling Security Fixes

- Record confirmed incidents in `SECURITYMD`.
- Link related runtime or engineering records in `RUNMD` and/or `ERRORMD`.
- Add preventive controls and validation steps before closure.
