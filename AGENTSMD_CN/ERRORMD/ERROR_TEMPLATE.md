# ERROR_TEMPLATE

## 使用说明

* ERRORMD 为日志模式（log），一条记录对应一次“系统更新/编译/工程流程”事件。
* 本模板只用于非运维错误与警告；运行时/运维问题应写入 RUNMD。
* 每次新增 ERROR 记录后，必须同步更新 `ERROR_INDEX.md`。

## Metadata

* **修改者：** [执行器 Agent 名称，例如 Codex]
* **事件时间（UTC）：** [YYYY-MM-DD HH:MM]
* **事件名（Key）：** [稳定事件名，例如 BUILD_COMPILE_FAIL]
* **分类：** [BUILD / COMPILE / UPDATE / DEPENDENCY / TEST / LINT / OTHER]
* **级别：** [FATAL / ERROR / WARNING / NOTICE / INFO / DEBUG]
* **影响范围：** [模块、目录、流水线或构建任务名称]
* **状态：** [OPEN / RESOLVED / MITIGATED]
